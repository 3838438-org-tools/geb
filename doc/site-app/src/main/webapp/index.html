What is it?

Geb (pronounced with a soft “G” like “Jeb”) is a browser automation solution for the Groovy programming language. It combines the browser driving features of WebDriver (and therefore can work with Internet Explorer, FireFox, Chrome and HTMLUnit) with a jQuery inspired content navigation/inspection API and the expressiveness of Groovy.

What is it good for?

Geb has a strong web testing focus (via it's first class support for JUnit, Spock Framework and EasyB) but is equally as good as a screen scraping or process automation tool.

What does it look like?

Here is some sample Geb…

import geb.*

Browser.drive("http://google.com/ncr") {
    assert title == "Google"

    // enter wikipedia into the search field
    $("input", name: "q").value("wikipedia")

    // wait for the change to results page to happen
    // (google updates the page without a new request)
    waitFor { title.endsWith("Google Search") }

    // is the first link to wikipedia?
    def firstLink = $("li.g", 0).find("a.l")
    assert firstLink.text() == "Wikipedia"

    // click the link 
    firstLink.click()

    // wait for Google's javascript to redirect 
    // us to Wikipedia
    waitFor { title == "Wikipedia" }
}

This is a very basic sample and does not illustrate many of Geb's most powerful features such as page objects, modules, the Content DSL and more. Please see the manual for more examples.

How can I use it?

Geb can be used standalone for scripting (like above) or in application code, or for testing. There is currently great support for testing with Geb via JUnit, Spock Framework and EasyB. There is also a Grails plugin (that works with JUnit, Spock or EasyB). Please see the “Integrations” section of the manual for details on using Geb with these libraries and frameworks.

How can I get help?

The manual is packed with information on how to use Geb. The community wiki contains higher level information about issues with certain drivers, links to blog articles or presentations and common design patterns.

You can also get help by asking questions on the mailing list.
