What is Geb?

Geb (pronounced with a soft “G” like “Jeb”) is a browser automation solution for the Groovy programming language. It combines the browser driving features of WebDriver (and therefore can work with Internet Explorer, FireFox, Chrome and HTMLUnit) with a jQuery inspired content navigation/inspection API and the expressiveness of Groovy.

What is it good for?

Geb has a strong web testing focus (via it's first class support for JUnit and Spock Framework) but is equally as good as a screen scraping or process automation tool.

What does it look like?

Here is some sample Geb…

import geb.*
  
Browser.drive("http://www.google.com") {
  assert title == "Google"

  $("input", name: "q").value("wikipedia")
  $("input", value: "Google Search").click()

  assert title.endsWith("Google Search")

  def firstResultLink = $("li.g", 0).find("a.l")
  assert firstResultLink.text() == "Wikipedia, the free encyclopedia"
}

This is very basic sample and does not illustrate many of Geb's most powerful features such as page objects, modules, the Content DSL and more. Please see the manual for more examples.

How can I use it?

Geb can be used standalone for scripting (like above) or in application code, or for testing. There is currently great support for testing with Geb via JUnit and via Spock Framework. There is also a Grails plugin which can work with either JUnit or Spock. Please see the “Integrations” section of the manual for details on using Geb with these libraries and frameworks.

How can I get help?

The manual is packed with information on how to use Geb. You can also get help by asking questions on the mailing list.
